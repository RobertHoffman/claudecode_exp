父亲策略文本里"主力操作"是**多层级**的——**大盘层面 / 个股主力层面 / 个股 K 线层面**，每一层语义不同。**绝不能把大盘的"出货/赶顶"语义直接套到个股上**。

**典型陷阱**（2026-06-24 PR-N Dual-Track 设计文档 v1.0-v1.4 全部踩过）：

- 父亲 V2 §三 第二阶段原文："借情绪跳空拉升...**借大盘顶部出货，'衰竭缺口'**"
- v1.0-v1.4 一直把"主力第二步"衰竭缺口""解读为"**主力高位出货**"——**错**
- 用户当面纠正："**赶顶是指大盘**，而要选中的股**不是顶部出货**，而是**提高价码吸筹**。做的隐蔽，**如果后期股价很快超越了基本上就证实了**。"
- 实质：
  - **大盘层面**：衰竭缺口 = 主力赶顶出货（**PR-N 指数 V 反负责检测**）
  - **个股主力层面**：借大盘冲顶掩护 = **提高价码隐蔽吸筹**（**本条件检测**）
  - **个股 K 线层面**：连续跳空 + 30d/200d 新高 + 光头光脚 = K 线形态（**PR-H "衰竭缺口识别"**）
  - **确认信号**：后期股价快速突破前高 = 主力吸筹成立

**Why**：人脑在处理"主力操作"这种跨层概念时，会**默认折叠所有层级到"个股主力"**——读到大盘"出货"就自动给个股也贴"出货"标签。但父亲实战经验里，**大盘在出货，个股主力在吸筹**——这是**对冲式操作**，大盘掩护下悄悄建仓。

**How to apply**：

1. **强制画"层级图"**（实施前必做）：
   - **Layer 1** 大盘层：regime 决定（PR-N INDEX_WASHOUT_REVERSAL、PR-M 浪型、PR-A BULL_EMOTION）
   - **Layer 2** 个股主力层：意图判定（吸筹 / 洗盘 / 拉升 / 出货 / 借势）
   - **Layer 3** 个股 K 线层：形态检测（PR-H 衰竭缺口 / PR-J 多头部剔除 / PR-C 距 52w 高）
   - 每个条件**先标注它属于哪一层**，再描述它的作用

2. **代码注释强制标注层级**（v1.5 修订示例）：
   ```python
   # ❌ v1.4 错版：模糊层级
   "consecutive_gap",  # 主力第二步：连续跳空（PR-H）
   
   # ✅ v1.5 正版：明确层级 + 意图
   "consecutive_gap",  # 个股借大盘冲顶：连续跳空（PR-H，主力借势隐蔽吸筹）
   ```
   注释里必须包含"大盘"/"个股"+"出货"/"吸筹"等关键词，**强制语义对齐**

3. **文档强制"分层语义区分"段**（v1.5 §3.3 新增）：
   - "重要语义区分：'赶顶/出货'是**大盘**层面（PR-N 负责），'借势吸筹'是**个股**层面（本条件检测）"
   - 任何条件若同时涉及大盘和个股，**必须**显式拆开写

4. **白话版（非技术版）同步修正**（v1.5 修订 V5 白话版 §第二步）：
   - "**大盘层面**是冲顶出货的信号；**但对我们选中的个股**，是主力借大盘冲顶掩护、在更高价位隐蔽吸筹的形态"
   - 父亲看的版本和工程版本必须**语义完全一致**

5. **OOS 报告里也分层统计**（v1.4 升级方向）：
   - 大盘 regime 触发 vs 个股条件触发 分开报告
   - 避免"大盘出货 + 个股吸筹"被合并统计掩盖

**类似陷阱**：
- 风控规则："市场大跌时减仓"——"市场"是 regime 层，"减仓"是仓位层，不能混
- 多策略组合："牛市做多"——"牛市"是 regime 层，"做多"是策略层，不能混
- 因子建模："动量因子"——是数据层（收益率），不能直接套"主力意图"语义
