多条件分类系统应**显式拆分"分类轴"（决定类别）和"仓位/打分轴"（决定权重）**，不要把不同维度的计数混在同一句话里。

**典型陷阱**（2026-06-24 PR-N Dual-Track 设计文档 v1.1/v1.2 连续踩过）：

- v1.1：写"6 关键 = MA5慢牛 + 连续跳空 + 30d高 + 光头光脚 + 200d高 + 5min≥2%"（6 个）
- v1.1 括号："200d高/光头光脚/5min 这 3 个非关键可软化"
- 用户 v1.1 抓出：'那 MA5 慢牛/连续跳空/30d 高 这 3 个能不能算进"失败 ≤ 2 个"的配额里？'
- v1.2 修法：拆"分类轴"（6 关键里 ≤ 2 失败 → TRACK_2）+ "仓位轴"（3 非关键里 0/1/2 → 0.50/0.40/0.30）
- **v1.2 仍错**：把 MA5 慢牛/连续跳空/30d 高 **混在"6 关键"失败配额池里**——这 3 个是 V2 主力四步法的**核心证据**（"主力在不在"），不能软化
- v1.2 隐含 bug：MA5 慢牛失败 + 5min 不达 → TRACK_2 半仓买入"无主力吸筹迹象"的票，违背体系起点假设
- **v1.3 根因**：我没有按"语义权重"分层，而是按"数量"分（"6 + 3"）。命名层"6 关键 + 3 非关键"是子集重叠（3 非关键 ⊂ 6 关键），父集叫关键、子集叫非关键——自我矛盾

**Why**：设计分类规则时容易把"决定类别"和"决定权重"两层概念混在一句话里，因为它们共用同一组条件。**但"哪些条件可以失败"和"哪些条件影响权重"是两个不同维度**，必须显式拆分。

**更深的根因**：v1.2 还没解决"语义权重"分层。**核心证据**（"主体在不在"）和**强化项**（"主体强不强"）是**完全不同的失败后果**——核心证据失败 = 主体不存在（SKIP），强化项失败 = 主体信号弱化（仍可半仓）。把这两类放在同一个失败配额池里，无论怎么"声明"不变量，**代码层必然写成"任意 ≤ 2 失败"**，留下隐藏 bug。

**How to apply**：

1. **v1.3 强制：先按"语义权重"分层，再按"数量"分条件**：
   - **第 1 层 = 硬门 0 容忍**（"主体在不在" / "环境允不允许"）：任一失败 → SKIP
   - **第 2 层 = 硬门 0 容忍**（"核心证据有没有"）：任一失败 → SKIP
   - **第 3 层 = 软门 ≤ N 容忍**（"主体强不强" / "强化项缺几个"）：失败 0 → TRACK_1，失败 1-N → TRACK_2，失败 > N → SKIP
   - **绝不**把"硬门条件"和"软门条件"放进同一个失败配额池——代码层必然写成"任意 ≤ N 失败"

2. **v1.2 经验：绘制两轴表**（适用于"分类轴 vs 仓位轴"维度）：
   - 轴 1（分类轴）：决定 X 在 {A, B, C} 哪一类，失败配额多少
   - 轴 2（仓位/打分轴）：决定 weight/size_pct，档位多少
   - 即使共用条件集合（如 3 强化条件同时是仓位轴的子集），作用也**完全独立**

3. **代码层强制分离**：
   - `classify_track()` 第 1 步**先**检查硬门层，**任一失败直接 return SKIP**（不进入后续计数）
   - `classify_track()` 第 2 步检查软门层，**只数软门条件的失败数**
   - 硬门层和软门层**完全独立计数**，**永不**混在同一个 `failed_count` 变量里
   - 不要写 `if failed_count <= 2: return "TRACK_2"`（混合硬门软门），而要写 `if hard_layer_passed and soft_failed_count <= 2: return "TRACK_2"`

4. **命名层强制"互不重叠"**：
   - 硬门层和软门层**用不同前缀**命名（FOUR_MUST_CONDITIONS / THREE_MAIN_FORCE_EVIDENCE / THREE_REINFORCEMENT）
   - 绝不用"6 关键 + 3 非关键"这种"子集叫非关键"的自相矛盾命名
   - 命名必须**语义准确**：核心证据 vs 强化项，必备 vs 加分，必须 vs 可选

5. **文档层强制声明不变量**：
   - "硬门层失败 → 直接 SKIP，**不计入**任何配额"（v1.3 关键不变量）
   - "软门层失败 = 第 3 层失败数，与硬门层完全独立"
   - "代码层强制顺序：第 1 层 → 第 2 层 → 第 3 层，不可颠倒、不可合并"

6. **决策表覆盖**（实施前必画）：
   - 至少 10-12 行：全通过 / 软门单失败 / 软门双失败 / 软门全失败 / **硬门任一失败** / 硬门全失败
   - **必须包含硬门失败的 3+ 行**（每行 1 个硬门条件失败）——v1.2 的决策表漏了这 3 行，导致我以为"6 关键里任意 2 失败"是合法路径
