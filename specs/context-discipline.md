# 上下文管理纪律

> 上下文越长模型越蠢。关键不是 Prompt Engineering，是 Context Engineering。

---

## 决策矩阵

| 场景 | 操作 | 信号 |
|------|------|------|
| 全新任务 | `/clear` 或新会话 | 任务与当前上下文无关 |
| 方向错了 | `Esc×2` 回退到读文件后的节点 | 方案被否定，失败尝试污染上下文 |
| 上下文 70%+ 还要继续 | `/compact focus on <当前重点>` | 主动压，不等自动触发 |
| 需要大量中间探索 | **子代理** | 只需结论，不需要中间过程 |
| 同一问题纠正 >2 次 | `/clear` | 双重修正规则——上下文已被失败尝试污染 |
| 大型多阶段功能 | 面试模式 → spec → 每阶段新会话 | 避免早期误解扩散到后续阶段 |

---

## 双重修正规则

同一问题纠正超过两次 → 立即 `/clear`。
干净会话 + 更好提示 > 长会话 + 累积修正。

---

## 子代理隔离

子代理核心价值是**上下文隔离**——中间过程留在一次性上下文里，只返回结论。

判断标准：「之后还需要这些工具输出本身，还是只需要最终结论？」
- 只需要结论 → 子代理
- 需要保留过程 → 主会话

典型用法：
- 「用子代理调查 X 模块如何处理 Y」→ 只返回摘要
- 「用子代理验证实现是否符合 spec」→ 只返回发现
- 节省 40%+ 输入 token

---

## `/compact` 时机

- 主动压缩：上下文 70% 时就 `/compact focus on <主题>`
- 被动压缩（自动触发）：在模型最蠢的时候做最关键的总结 → 不可靠
- 压缩保留项（写进 CLAUDE.md 压缩指令）：修改文件列表、测试命令与结果、当前实现计划、未解决 error

---

## `.claudeignore`

阻止以下目录/文件进入上下文：
```
node_modules/
.venv/
__pycache__/
dist/
build/
*.lock
*.min.js
coverage/
*.png
*.jpg
*.pdf
output_data/
reports/
logs/
```

CLAUDE.md、PROJECT.md、specs/ 始终可访问。
