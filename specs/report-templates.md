# 报告模板 & 通知格式

> 阶段2（计划+执行）和阶段5（结果报告）执行前必读。

---

## 阶段2：需求 & 计划模板

```
🔥 【需求&计划已生成】请评审 @Hoffman

📋 范围
- 功能描述: ...
- 受影响文件: ...
- 分支: feature/task-xxx

📐 实现计划
1. ...
2. ...

⚠️ 边缘 Case
- ...
- ...

🧪 测试策略
- 单元测试: ...
- 关键路径回归: ...

🎯 验收标准
- [ ] ...
- [ ] ...

🚨 风险
- ...
```

→ **行动: [批准 / 暂缓 / 修改 / 补充信息]**
→ 用户回复"通过/批准/OK"= 写 `.claude/active-plan.md` **并** 立即委派 worker 执行
→ 用户回复"暂缓"= 只写 plan.md，**不** 委派 worker（后续可手动触发）
→ 用户回复"修改"= 重新规划
→ 用户回复"补充信息"= 等用户回答追问

---

## 阶段5：结果报告模板

### 标准模式（完整报告）

```
🔥 【结果报告已生成】请评审 @Hoffman

📊 变更总结
- 新增: N 文件, +M 行
- 修改: N 文件, +M/-K 行
- 删除: N 文件

✅ 测试结果
- 通过: N / 失败: M / 缺失覆盖: K
- (失败详情，如有)

🔍 审计摘要
- 致命: N  高: N  中: N  低: N
- (致命/高问题逐条列出)

🚨 风险列表
- [ ] 高风险: ...
- [ ] 中风险: ...

📝 演示步骤
1. ...
2. ...

⏮️ 回滚计划
- git revert <commit> 或
- 切回 feature branch 前状态: ...

📁 关键文件
- (变更文件列表)
```

### 快速模式（简化报告）

```
✅ 完成: (一句话变更描述)
测试: N passed / M failed
风险: 无 / (简述)
```

---

## 通知行动标签

所有 🔥 标记消息末尾必须包含行动标签：

| 标签 | 含义 |
|------|------|
| `→ 行动: 批准` | 等待用户说"通过/批准/OK" |
| `→ 行动: 修改` | 需要用户指明修改方向 |
| `→ 行动: 补充信息` | 需要用户提供更多上下文 |

---

## 通知触发点

| 阶段 | 触发 | 格式 |
|------|------|------|
| 阶段2 完成 | 计划输出后 | `🔥 【需求&计划已生成】请评审 @Hoffman` |
| 阶段4 完成 | 审计报告后 | `🔥 【审计报告】请评审 @Hoffman` |
| 阶段5 完成 | 结果报告后 | `🔥 【结果报告已生成】请评审 @Hoffman` |
| 需要外部干预 | 阻塞/风险 | `🔥 【需要外部干预】... @Hoffman` |
| 快速模式完成 | 简化报告后 | `✅ 完成: ...`（无 🔥，无需等待） |
