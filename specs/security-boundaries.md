# 安全边界

> 硬约束。违反任一条 → Driver 拒绝执行并通知用户。

---

## 禁止 AI 自行处理的领域

| 领域 | 规则 | 触发条件 |
|------|------|---------|
| **认证/授权** | 禁止自行修改 auth 逻辑、token 生成/验证、OAuth 流程、session 管理 | 改动涉及 `auth/`、`login/`、JWT、OAuth |
| **加密** | 禁止自行实现/修改加密算法、密钥派生、证书生成 | 改动涉及 `crypto/`、`encrypt/`、AES/RSA/hash |
| **凭证/密钥** | 禁止生成、存储、轮换 API Key/Token/密码 | 改动涉及 `.env`、`credentials`、`secret` |
| **数据库 Schema** | 禁止执行 DROP/ALTER TABLE、删除 collection，只能生成脚本待审批 | `DROP`、`ALTER`、`remove()` |
| **财务/交易** | 禁止修改金额计算、手续费率、盈亏公式 | 改动涉及 `pnl/`、`fee/`、`order/` |
| **权限/角色** | 禁止修改 RBAC/ACL、用户角色提升 | 改动涉及 `role/`、`permission/`、`admin/` |

---

## 执行机制

| 层级 | 机制 |
|------|------|
| **人工门禁** | 上述变更必须经过阶段2（计划）+ 用户明确批准 + 阶段5（报告）+ 用户二次确认 |
| **CLAUDE.md 硬约束** | Driver 识别到上述领域 → 强制进入标准模式，禁止快速模式 |
| **PreToolUse Hook**（可选加强） | 匹配到敏感路径时 `exit 2` 阻塞，提示人工评审 |

---

## Driver 识别流程

收到任务后：
1. 判断是否触及上述领域
2. 如果触及 → 回复：「⚠️ 此任务涉及 [领域]，按安全边界规则，必须走完整 6 阶段流程 + 人工评审。不可快速模式。」
3. 如果用户坚持快速 → 拒绝：「安全边界规则禁止快速模式处理此类变更。」
